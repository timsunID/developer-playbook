# Install fnm

## Purpose

Install Fast Node Manager (fnm) to manage Node.js versions consistently.

## When to use it

- Fresh machine setup
- Standardizing Node version management across projects

## Prerequisites

- Terminal access
- Internet connection

## Steps

1. Install fnm using the official install method for your OS.
2. Ensure shell integration is configured.
3. Restart terminal.

## Verify it works

```bash
fnm --version
```

## Common mistakes

- Installing fnm but not enabling shell integration
- Running commands in an old terminal session

## Notes / best practices

- Use fnm as the single source of truth for local Node versions.
